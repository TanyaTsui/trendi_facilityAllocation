Reading and cropping rasters¶

In this notebook, we convert the rasters downloaded from the Annual Crop Inventory of Canada into a shapefile of points, showing the ACI code at each location. Each ACI code represents a type of landuse for agriculture, such as 'greenhouses', 'oats', 'too wet to be seeded', 'wheat'...etc. Then, we ensure that the points are correctly categorized by province.

# extract geoms from rasters
shps = dict.fromkeys(['ab', 'bc', 'mb', 'nb', 'nl', 'ns', 'on', 'pe', 'qc', 'sk'])

for province in shps.keys(): 
    # print('extracting geoms from {} raster...'.format(province))
    mask = None
    with rs.Env():
        with rs.open('data/tif/3000m_aci_2020_{}_class.tif'.format(province)) as src:
            image = src.read(1) # first band
            results = (
            {'properties': {'raster_val': v}, 'geometry': s}
            for i, (s, v) 
            in enumerate(
                shapes(image, mask=mask, transform=src.transform)))
    geoms = list(results)

    # make shpfile, formatting
    gdf_fromGeom = gpd.GeoDataFrame.from_features(geoms)
    gdf_fromGeom.rename(columns={'raster_val': 'aciCode'}, inplace=True)
    gdf_fromGeom.aciCode = gdf_fromGeom.aciCode.astype('int64')
    gdf_fromGeom = gdf_fromGeom[gdf_fromGeom.aciCode > 0] # remove non-agricultural cells
    
    # add to shps dictionary 
    gdf_fromGeom['province'] = province
    shps[province] = gdf_fromGeom

aci = pd.concat(shps.values())
aci.geometry = aci.geometry.centroid
aci = aci.set_crs(crs_wkt)

Categorise points by province¶

The raster files provided by the Annual Crop Inventory (ACI) do not completely follow the province borders. In order to ensure that waste yield are correctly estimated in the next notebook 01_dataWrangling_2_calcYield, we need to ensure that the points in the aci geodataframe are assigned to the correct province.

In [54]:
%%time 

# define function for filtering points from aci according to province 
def filterPts(province): 
    canada_p = canada[canada.province == province]
    aci_p = aci[aci.province == province]
    aci_p['withinP'] = aci_p.geometry.within(canada_p.iloc[0, -1])
    aci_p = aci_p[aci_p.withinP == True]   
    return aci_p

# filter points for each province 
provinces = ['ab', 'bc', 'mb', 'nb', 'nl', 'ns', 'on', 'pe', 'qc', 'sk']
toConcat = []

for province in provinces: 
    start = time.time()
    pts = filterPts(province)
    end = time.time()
    toConcat.append(pts)
    print('time taken for {}: {}s'.format(province, round(end-start)))
    
aci_all = pd.concat(toConcat)

Limitations and further work¶

Limitations

  • Rasters (geotiffs) from the annual crop inventory were downsampled from 800m to 3km resolution, to decrease file size and runtime. Because the resolution is lower, the locations of agricultural waste is less accurate. This is because some information is inevitably lost in the data aggregation process.
  • After downsampling, the rasters were then converted into points, to further decrease the file size and runtime. This further lowers the accuracy of the map of agricultural waste.
  • The resolution of 3x3km is acceptable for this country-wide analysis, because the size of Canada is around 4000 x 3000 km. In fact, when doing a country-wide analysis, the resolution could probably be even lower then 3km.

Further work (optional, to improve accuracy of results)

  • We need to check how downsampling the rasters from 800m to 3km resolution affects the accuracy. Arjang has already checked this for 2km resolution. So we should either check the quality of the rasters at 3km resolution, or use 2km resolution rasters.
  • Perhaps even lower resolution rasters could be used, because this is a country-wide analysis. This depends on how much information is lost during the downsampling process.
